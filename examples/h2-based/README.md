## Project Structure

This is a Rust HTTP server built using AWS Smithy's code generation toolchain. The project consists of two main crates:

- `rust-server-codegen/` - Generated Smithy server code providing the HTTP framework
- `service/` - Application implementation with business logic handlers

## Common Commands

### Build and Run
```bash
# Build the entire workspace
cargo build

# Run the service (starts HTTP server on port 9090)
cargo run -p check-h2

# Build specific crate
cargo build -p server  # rust-server-codegen crate
cargo build -p check-h2  # service crate
```

### Testing
```bash
# Run all tests
cargo test

# Test a specific operation endpoint
curl -X POST http://localhost:9090/sample \
  -H "Content-Type: application/json" \
  -d '{"inputValue": "test value"}'
```

## Architecture

The service uses AWS Smithy's generated code to create a Tower-based HTTP server:

1. **Entry Point**: `SampleService` in `rust-server-codegen/src/service.rs:449` implements `tower::Service`
2. **Routing**: `RoutingService` routes HTTP requests to appropriate handlers
3. **Operations**: Individual operations (like `SampleOperation`) handle business logic
4. **Serialization**: Automatic JSON serialization/deserialization via Smithy protocols

## Key Files

- `service/src/main.rs` - Application entry point and handler implementations
- `rust-server-codegen/src/service.rs` - Generated service and routing logic
- `rust-server-codegen/src/input.rs` - Generated input types
- `rust-server-codegen/src/output.rs` - Generated output types

## Handler Implementation

Handlers in `service/src/main.rs` must match the signature:
```rust
async fn handler(input: SampleOperationInput) -> Result<SampleOperationOutput, SampleOperationError>
```

The generated code handles HTTP routing, request parsing, and response serialization automatically.

## Generated Code

The `rust-server-codegen/` crate is entirely generated by Smithy toolchain. Do not manually edit files in this crate - they will be overwritten during regeneration.
