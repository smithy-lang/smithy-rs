Register dual RPC v2 CBOR routes by default for operations whose names differ between the verbatim Smithy operation name and the PascalCased Rust symbol name. Both the spec-compliant verbatim route (e.g., `getFoo`) and the legacy PascalCased route (e.g., `GetFoo`) are registered, providing backward compatibility while fixing client/server interoperability. Operations already in UpperCamelCase (where names match) get a single route as before.

Add opt-out server codegen setting `rpcV2CborExcludeLegacyOperationNameRoute` that drops the legacy PascalCased route, registering only the spec-compliant verbatim route. Use this once clients have migrated to the correct URIs.
